🔑 GitHub API Token 设置指南

⚠️ 重要提示：GitHub API Token 是跨设备同步功能的核心。没有正确设置 Token，就无法在云端保存和同步数据。

步骤 1：创建 GitHub API Token

1. 访问 GitHub 设置页面：

https://github.com/settings/tokens

2. 点击 "Generate new token" → "Generate new token (classic)"

3. 填写以下信息：

  • Note (备注)： 购销单系统同步
  • Expiration (过期时间)： 选择 "No expiration" 或 "90 days"
  • Select scopes (权限)： 勾选 gist 权限

4. 点击页面底部的 "Generate token" 按钮

⚠️ 重要：Token 生成后只会显示一次，请立即复制保存！

🚀 快速测试同步功能

设置完 Token 后，您可以：

  1. 在 PC 端生成同步码
  2. 在手机端输入同步码进行同步
  3. 使用测试页面验证功能

常见问题解决

❓ Token 无效或过期

  • 检查 Token 是否正确复制（包含 "ghp_" 前缀）
  • 确认 Token 没有过期
  • 确认勾选了 "gist" 权限

❓ 同步仍然失败

  • 确保 PC 端和手机端使用相同的 Token
  • 检查网络连接
  • 尝试重新生成同步码

❓ 无法访问 GitHub

  • 检查网络连接
  • 尝试使用 VPN 或代理
  • 确认防火墙设置
✅ 设置完成后：
1. PC 端和手机端都会自动使用保存的 Token
2. 生成的同步码可以在任何设备上使用
3. 数据会安全地保存在您的 GitHub Gist 中
